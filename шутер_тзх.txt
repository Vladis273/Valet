Сюжетный шутер: 

--Игрок--
	Стрельба: 
		- (+) Источник выстрела: геометрия оружия (не камера), что повышает реализм прицеливания и анимаций.
		- (+) Режимы огня: одиночный / очередь (длина настраивается) / автомат — переключаются через клавишу и дает текстовую подсказку.
		- Система попаданий:
			(+) Основное оружие - сканхит (hit-scan)
			Снайперские винтовки, гранатомёты - проджектайлы
		- Точность и разброс:
			(+) Первые N выстрелов (настраивается) всегда идут точно в точку прицеливания, независимо от движения, прыжков или позы.
			(+) После этого начинает накапливаться круговой разброс, растущий с заданной скоростью до максимума.
			(+) При прекращении стрельбы разброс постепенно уменьшается (время восстановления настраивается).
			(+) Перезарядка сбрасывает разброс мгновенно.
		- Прицеливание:
			(+) Увеличивает количество «гарантированно точных» выстрелов.
			(+) Может замедлять движение (опционально).
			*Если будут реализованы модули, то лазерный прицел (в будущем - лцу) будет создавать лазерную точку в центр экрана (У игрока нет стандартного прицела в UI), при прицеливании с лцу улучшает некоторые параметры стрельбы.
			
		* Будущее расширение: система модулей (обвесов) будет влиять на параметры разброса, скорость восстановления, точность в движении и т.д.
	
	Взаимодействие с оружием: (Требуется: Инвентарь, Противники и Команда нпс)
		- Подбор и замена (Требуется: Инвентарь)
			(+) Игрок может нести 2 единицы оружия.
			(+) Замена/подбор происходит удержанием кнопки перезарядки при наведении на оружие на земле.
			(+) Если слот свободен — оружие подбирается.
			(+) Если оба слота заняты — текущее оружие заменяется.
			(+) Нельзя выбросить оружие вручную.
			(+) Подобранное оружие содержит только ту обойму, что была вставленной у бота (может быть частично или полностью пустой).
		- Система магазинов
			Для каждого типа оружия хранится отдельный пул магазинов (до 3 типов одновременно).
			При перезарядке выбирается обойма с наибольшим числом патронов.
			Пустые магазины утилизируются, неполные — сохраняются.
			Максимальное число магазинов фиксировано (например, 4 для АК, 2 для пистолета) и нельзя превысить, даже если найдены новые.
			Умная замена: если все слоты магазинов заполнены, но есть неполные — при подборе нового магазина заменяется самый пустой.
		- Восполнение боекомплекта
			Основной способ — запрос поддержки: (Требуется Команда нпс) союзный НПС сбрасывает сумку с патронами, полностью восстанавливая лимит магазинов.
			Дополнительно — (Требуется Противники) сбор магазинов только подходящего типа с тел убитых ботов.
		- Проверка боекомплекта
			(+) Удержание R без наведения на оружие → анимация осмотра магазина + текстовая подсказка (без поцентов):
				- Full (100%)
				- Almost full (99–61%)
				- Half-full (60–40%)
				- Almost empty (39–1%)
				- Empty (0%)
			(+) При перезарядке, по окончанию показывается подсказка (без процентов)
			
	Передвижение:
		- Базовое:
			(+) Ходьба / бег (ускорение при удержании Shift (По умолчанию)).
			(+) Бег вбок возможен только при одновременном удержании W + A/D (По умолчанию) → иначе — ходьба.
			(+) Прыжок (Space).
			(+) Три позы: (+) стоя → присяд → лёжа (только одна активна). 
			Ограничения:
				- Пробел из положения «лёжа» → «присяд»; из «присяд» (если удерживается) → «стоя».
				- В присяде/лёжа нельзя прыгать, в прыжке — нельзя менять позу.
		- Движение рук: 
			(+) Динамические руки: руки будут следовать с задержкой за камерой, при движении, прыжках, приседаниях и т.д. - это будет имитировать поворот, движение, падение и т.д. 
			*Является балансировкой точной стрельбы игрока.
		- Вертикальное перемещение:
			(+) Лестницы, решётки — стандартный climb.
			Уступы/ящики: можно залезть, только если центр масс игрока выше половины объекта.
			Командная помощь: по запросу НПС может подсадить игрока на уступ (если технически возможно). Иначе — отказ.
		
	Командное взаимодействие:
		- Вызов радиального меню → выбор из 3 способностей, соответствующих составу отряда.
		- Возможности зависят от наличия НПС определённой специализации.
		
	Система повреждений и выживания:
		- Регенерация HP начинается после N секунд без урона (настраивается).
		- Урон по частям тела: голова ×1.5, ноги ×0.7 и т.д.
		- Лечение по запросу: медик восстанавливает 100% HP, но отказывает, если HP > 75%.
		- При смерти:
			Игрок падает и ждёт медика.
			Если медик мёртв / не успел / игрок добит — смена тела на одного из живых союзников.
				Приоритет: штурмовик - инженер - поддержка - медик (последний — чтобы не терять функцию воскрешения).
			Если союзников нет — возврат на последнюю контрольную точку.
			
	Инвентарь:
		- 2 слота оружия
		- (+) 4 типа гранат, количество настраивается
			(+) типы гранат: осколочная/светошумовая/зажигательная/дымовая
		- 3 типа магазинов (два для текущих в слотах + один если игрок решит вернуть прошлое оружие):
			При смене оружия старые магазины сохраняются, пока не будут заменены новыми.
		
--Команда (Союзные НПС)--
	- Управляются единым многопоточным менеджером (для производительности).
	- Передвижение через NavMesh с укрытиями и перебежками.
	- Действуют автономно: занимают укрытия, перебегают, не «прилипают» к игроку.
	- Пытаются покинуть зону огня.
	- Не мешают: если игрок идёт в НПС — тот уходит в сторону.
	- В спокойной обстановке — держатся рядом; если не могут добраться до игрока — занимают оборону.
	- Специализация:
		|  	Класс  		|	Способности																															|
		|	Штурмовик	|	Стандартный боец. Имеет стимулятор: +10% HP, −25% урона, −30% время до регена (длительность настраивается).							|
		|	Медик    	|	Лечит/поднимает союзников. Не лечит, если HP > 75%. Не может поднять себя. Последний в списке смены тела.							|
		|	Инженер  	|	Носит гранатомёт (3–4 выстрела). По запросу — стреляет по указанной точке. Может кидать гранаты.									|
		|	Поддержка	|	Носит сумку патронов. Автоматически восполняет 1 магазин игроку (частота настраивается). По запросу — сбрасывает сумку на точку.	|
	- Все союзные НПС получают в 2 раза меньше урона, чем игрок (компенсация за ограниченный ИИ).
	- Имеют бесконечный боезапас магазинов, но при смерти имеют шанс на подбор 1 - 2х магазинов с тела.
	* Продвинутый ИИ - отложен до завершения базовой версии. Включает логику ботов, новые команды: атаковать выделенных ботов, переместиться на точку и т.д.
	
--Противники--
	- Управляются единым многопоточным менеджером (для производительности).
	- Передвижение через NavMesh с укрытиями и перебежками.
	- Поведение:
		Не регенируют HP и не может быть поднят (0 хп - сразу смерть).
		Всегда имеют максимальный разброс → исключает «снайперский» мгновенный урон.
		Стреляют по игроку при обнаружении; если скрыт — могут обстреливать позиции союзников.
		Пытаются покинуть зону огня.
		Могут кидать 1–2 гранаты за бой (не все типы).
		При укрытии игрока — начинают подход.
	- Специализация:
		| Класс	    |	Особенности																																			|
		| Штурмовик |	Агрессивный подход, гранаты, светошумовые.																											|
		| Инженер   |	Гранатомёт (1–2 выстрела) или особая граната: зажигательная/дымовая																					|
		| Снайпер   |	Дистанционный бой, без разброса, но с задержкой выстрела. При приближении — переключается на пистолет. (Продвинутый ИИ: залезает на возвышенности)	|
		| Поддержка |	Подавляющий огонь, больше HP, получает меньше урона.																								|
		| Медик	    |	Самолечение, кидает светошумовые. (Продвинутый ИИ): Лечит раненых ботов за укрытием. 																|
	- Имеют бесконечный боезапас магазинов, но при смерти имеют шанс на подбор 1 - 2х магазинов с тела.
	* Продвинутый ИИ — отложен до завершения базовой версии. Включает координацию, тактические группы (1–2 атакуют, остальные прикрывают), использование высот (Использование лестниц или уступов).

--Механики оружия и гранат--
	Оружие:
		Все снайперские винтовки или дмр винтовки имеют проджектайлы пули с баллистикой.
		Некоторое оружие способно прошивать или разрушать стенки или планки из-за своего калибра, при простреле урон теряется (Настраивается).
		Перезарядка оружия с пустым патронником происходит медленнее, чем с полным.
	Гранаты:
		- Осколочная: обычный взрыв, чем дальше от центра, тем меньше урона.
		- Светошумовая: Ослепляет и оглушает, если взорвалась когда игрок смотрит на нее, иначе вспышка со стороны гарнаты и небольшой гул в ушах. Станит ботов.  
		- Дымовая: Тушит огонь зажигательной гранаты. ограничивает видимость внутри. боты считают дым укрытием, но могут видеть, если близко к игроку.
		- Зажигательная: наносит переодический урон + урон от горения на N секунд (настраивается), поджигает всех.
		
--Тактическая среда--
	Разрушаемые: 
		Декоративная (стулья, бутылки) — только визуал.
		Тактическая (окна, фонари) — влияет на обзор/звуки.
	Простреливаемые:
		Слабые укрытия (фанера, столы) — пропускают пули с потерей урона.
		Бронированные (металл, бетон) — не простреливаются.
	Огонь и дым:
		Зона горения: Боты обходят зону и пытаются выйти если попали в нее.
		Дым блокирует ИИ-обзор, но не полностью (боты «чувствуют» игрока на близкой дистанции).
		* Продвинутый ИИ - Боты не будут кидать зажигательную гранату, если он полетит в дым и потушится

--Мелкие моменты--
	Система состояний персонажа:
		- Состояния: 
			Игрок может одновременно находиться в разных состояниях:
				Поза (стоя / присев / лёжа)
				Движение (идёт / бежит / ползёт / прыгает)
				Действие с оружием (стреляет (и/или) прицеливается / перезаряжает).
			Известные ограничения:
				Нельзя стрелять в движении лежа (Пример: ползти в бок и стрелять).
				Бежать и стрелять - игрок переходит в ходьбу
				Нельзя перезаряжаться в воздухе.
				При смене позы (например, лёжа → присев) стрельба не прерывается, если новая поза допускает огонь.
				Осмотр магазина может быть прерван только перезарядкой.
				*В состоянии «повержен» (Downed) игрок или бот может медленно ползти, но не может использовать оружие.
		- Переходы между состояниями должны быть строго определены.
		- Анимационные события: триггеры для звука выстрела, начала/конца перезарядки, смены позы.
	Система наведения и цели:
		- Курсор / точка прицеливания: фиксированная точка в центре экрана отсутствует (Если нет лазерного прицела (см. стрельба - прицеливание)).
			Игрок сам должен определять центр или использовать прицеливание.
		- Raycast / Spherecast для ADS: проверка, куда смотрит игрок, чтобы активировать подсказку «можно подобрать» или «наведён на НПС для команды».
	Система предметов и дропа
		- Дроп с ботов: при смерти бот генерирует объекты с заданными шансами: 
			Оружие: Всегда падает активное оружие бота.
			Магазины: При подборе с тела бота дает от 1 до 2 (настраивается) магазина активного оружия + 0 - 1 дополнительного (при наличии специализации и оружия во втором слоте).
			Гранаты: Зависит от специализации, имеет шанс от 0 до 1 шт (если у бота может быть 2 вида гранат, выбирается случайно).
		- Зона подбора: радиус, в котором предмет автоматически подбирается (в основном гранаты, если есть свободный слот и они лежат на полу (не требуют подбора с тела)) или становится интерактивным.
	Система эффектов и статусов:
		- Базовые статусы:
			Игрок: Regenerating, Burning, Stunned и Blinded (оглушение и ослепление от светошумовой).
			Боты: Regenerating (только союзные), Burning, Stunned и Blinded.
		- Визуальные/звуковые индикаторы: экран краснеет при низком HP, оранжевый экран + частицы огня.
			Светошумовая: белый экран и звенение в ушах при прямом ослеплении (светошумовая сработала в обзоре игрока) или осветление экрана или небольшой гул (игрок отвернулся).
		- Таймеры и сброс: горение длится N секунд, стан — 2 секунды и т.д.
	Базовая система настроек:
		- Input Action Map (Unity Input System или аналог): R (нажать) = Reload, R (зажать) = Inspect/Pickup.
		- Настройки чувствительности, инверсии мыши, яркости, субтитров.
		- Сохранение настроек между сессиями.
		* Некоторые механики имеют несколько действий на одну клавишу, нельзя делать жесткую привязку, а требуют контекстной обработки ввода.
	Система событий и сообщений:
		- События: Много взаимодействий между системами не происходящих постоянно.
		- Подписка компонентов на события без жёстких ссылок.
	Система смены тела для игрока и ИИ:
		- При смерти игрок меняет тело на доступного союзного бота (Вытесняет ИИ из тела).
		- При возрождении умершего союзника, его будет брать под контроль ИИ, игрок остается управлять телом после переноса 
			(Если игрока перенесло в поддержку, то после возрождения штурмовика (начальный класс), игрок останется в поддержке до след. смерти , а штурмовика возьмет ИИ).
		- Игрок не может менять тела самостоятельно.

--Стандартное управление--
	Передвижение:
		Вперед: W
		Назад: S
		Влево: A
		Вправо: D
		Бег: Left Shift
		Прыжок: Space
		Поза вприсядь: (Удерживать: да/нет) C
		Поза лежа: Z
	
	Оружие:
		Выстрел: Mouse1
		Прицеливание: Mouse2
		Слот 1: 1
		Слот 2: 2
		
		Перезарядка: R
		Замена оружия (Удержать при наведении): R
		Осмотр магазина (Удержать без наведения): R
		
		Сменить темп стрельбы: B
		Открыть окно модулей (Будущее): X

	Граната:
		Бросок гранаты: G
		Сменить тип гранаты: V
	
	Команда:
		Вызов радиального меню: Tab
		Активировать способность (стимулятор и др.): F
		
	Прочее:
		Меню: Esc
		Взаимодействовать: E

--Технические требования--
	- Поэтапная реализация: сначала ядро (движение + стрельба + базовый ИИ), затем слои (команда, модули, продвинутый ИИ).
	- Настройки через редакторы: скрытие сложных параметров от обычного пользователя.
	- Производительность:
		Минимизация нагрузки на главный поток.
		Многопоточность для управления группами юнитов.
		Дополнения (*.unitypackage):
			Frustrum Culling - для оптимизации декораций и мелких объектов за обзором игрока.
			Dynamic Mesh Combiner - оптимизация объединением большого кол-ва объектов в один.
			Dynamic Bone - облегчение создания анимаций для персонажей через "кости".
